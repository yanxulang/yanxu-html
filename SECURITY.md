# 安全策略

## 支持版本

| 版本线 | 安全修复 |
| --- | --- |
| 1.0.x | 支持 |
| 0.1.x | 1.0.0 发布后不再支持 |

安全修复会发布新的补丁版本，并在更新记录与 GitHub Security Advisory 中说明受影响版本和迁移方式。

## 私下报告

请使用仓库的 GitHub Security Advisory 提交安全报告，不要在公开 issue、讨论、拉取请求或分支中附带可利用载荷。报告至少应包含：

- 受影响的言页和言序版本；
- 最小节点构造或输入；
- 实际输出与预期安全边界；
- 可利用上下文及影响；
- 已知绕过、缓解方式或修复建议。

维护者会在私下渠道确认问题、复现影响并协调修复与披露。不要提交令牌、Cookie、个人数据或真实生产地址。

## 范围

下列行为通常属于安全问题：

- `文字`或文字属性未正确转义；
- 默认策略可通过大小写、控制字符或重复属性绕过；
- URL 属性接受策略外协议；
- 资源预算可被绕过并造成无界输出；
- 错误意外泄露不应公开的输入或运行信息。

`原始`和`扩展`是明确的可信接口。把不可信内容交给它们，或让写出器自行拼接不可信 HTML，不属于默认转义绕过；若可信接口能够逃逸配置的字符或片段预算，则仍属于安全问题。

完整威胁模型见 [docs/SECURITY_MODEL.md](docs/SECURITY_MODEL.md)。
