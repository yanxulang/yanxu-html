# 言页架构

## 职责

言页只负责把显式 HTML 节点图转换为受预算约束的文字片段。它不解析模板文件、不读取数据、不监听网络、不清洗已有富文本，也不决定 HTTP 状态和首部。

核心模块没有第三方依赖，只使用言序标准库的文字、模板、正则、字节和数学能力。

## 分层

```text
应用数据
  │
  ├─ 普通路径：文字 / 属性 / 元素 / 片段 / 组件 / 文档
  │                │
  │                └─ 构造期名称、类型、URL、重复项与空元素检查
  │
  └─ 可信路径：原始 / 扩展
                   │
                   └─ 显式信任升级，不做清洗

节点图
  │
  └─ 统一递归渲染器
       ├─ 深度、节点、属性预算
       ├─ 文字与属性上下文转义
       ├─ 字符、片段预算
       └─ 同步写入回调
             ├─ 文本收集器 -> 完整正文
             └─ 调用方写入器 -> 增量输出
```

完整渲染和流式渲染复用同一个节点遍历器。前者只是把写入回调绑定到私有文本收集器，因此两条路径不会形成不同的安全实现。

## 节点模型

`HTML节点`是公开根类，具体节点承担不同职责：

| 节点 | 内容 | 渲染行为 |
| --- | --- | --- |
| `HTML文字节点` | 普通文字 | HTML 转义后写出。 |
| `HTML原始内容` | 可信文字 | 原样写出，仍经过预算。 |
| `HTML扩展节点` | 私有写出器 | 把受控写入法交给可信生成器。 |
| `HTML片段` | 节点列 | 顺序展开，不增加标记。 |
| `HTML元素` | 标签、属性、子节点 | 写开始标签、子节点和闭合标签。 |
| `HTML组件` | 无参构建器 | 每次渲染构建并验证一个节点。 |
| `HTML文档` | 根节点 | 写 HTML5 doctype 后渲染根。 |

`HTML属性`不是节点；它在元素渲染时输出，并受每元素属性预算约束。

## 所有权与快照

言序列是可变值。元素和片段若直接保留调用方传入的列，调用方可在构造后绕过重复属性、类型和空元素检查。因此构造器逐项验证并复制输入列。

`各属性`和`各子节点`也返回新列，避免读取操作泄露内部可变状态。受控变更必须通过`添属性`或`添子`，由同一不变量检查保护。

节点对象本身仍是构建期对象，追加操作不是并发同步原语。跨任务共享时应先完成构造，之后按只读节点图使用。

## 构造期安全检查

构造器尽早拒绝：

- 不合法或策略禁止的标签与属性名；
- URL 属性中的空白、控制字符和策略外协议；
- 大小写不敏感的重复属性；
- 属性列中的非`HTML属性`值；
- 子节点列中的非`HTML节点`值；
- 空元素的子节点；
- 超过直接子项或属性硬上限的输入列。

URL 属性在渲染时再次检查。这保持属性对象在未来扩展或反射场景下的最终输出边界。

## 渲染状态

每次渲染创建独立的私有状态，保存规范化限制、写入回调和三个计数器。

节点访问顺序：

1. 检查当前深度并增加节点数；
2. 按节点实际类型分派；
3. 元素在输出前检查属性数量；
4. 每个非空文字片段在交给调用方前检查字符与片段预算；
5. 成功完成后返回节点数、字符数和片段数。

根节点深度为 0。文档、组件和片段本身均计为节点；组件构建出的节点另行计数。属性不计入节点数。

字符数按`文字.字符列`计数，不按 UTF-8 字节计数。HTTP 层若还需要字节上限，应在编码或传输层另行限制。

## 完整与增量输出

`安全渲染`等价于`渲染受限（节点，{}）`。`渲染受限`使用文本收集器联结全部片段，只有成功后才把正文返回调用方。

`流式渲染`直接同步调用外部写入法。它不提供异步背压、取消、重试或事务回滚。若第 N 个片段写出后发生错误，前 N 个片段仍属于外部系统状态；渲染器只停止后续调用。

写入回调应快速、确定且不递归渲染同一无限结构。需要异步发送时，上层可把片段复制进有界队列，并由自己的背压协议消费。

## 可信扩展

`原始`是短文本信任升级，适合已审计常量或专用清洗器的最终输出。

`扩展`保存私有写出器。渲染时只向写出器暴露受预算约束的写入法，而不暴露状态、内部收集列或不受限输出入口。扩展可以输出任意可信 HTML，因此其安全责任与`原始`相同，但分片全部进入统一计数。

## 错误边界

内部错误统一使用`HTML_代码：消息`。`错误详情`只在消息确实具有此前缀时提取代码，否则归为`HTML_RUNTIME`。

错误代码属于兼容协议；中文消息、源位置和踪迹只用于诊断。应用不应把完整踪迹直接返回不可信客户端。

## 复杂度

在无扩展回调额外工作的情况下：

- 构造元素为 O(A + C)，A 为属性数、C 为直接子节点数；
- 渲染为 O(N + A + U)，N 为访问节点数、A 为全部属性数、U 为输出字符数；
- 完整渲染额外保存 O(U) 的片段与正文；
- 流式渲染除递归深度和调用方行为外，不保存完整正文。

当前遍历使用递归实现，深度预算同时限制调用栈增长。性能负载和调优原则见 [PERFORMANCE.md](PERFORMANCE.md)。
