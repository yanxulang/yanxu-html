# 言页安全模型

言页的目标是让服务端 HTML 构造中的常见注入错误难以意外发生。它提供安全默认原语、显式可信边界和资源预算，不替代业务授权、完整富文本清洗、浏览器隔离或 HTTP 安全策略。

## 输入分类

| 输入来源 | 应使用 | 行为 |
| --- | --- | --- |
| 用户、数据库、接口、文件 | `文字`或文字属性 | HTML 转义，URL 属性另行校验。 |
| 布尔状态 | 布尔属性 | `真`输出名称，`假`或`空`省略。 |
| 结构化页面 | `元素`、`片段`、`组件`、`文档` | 名称、类型、组合和预算检查。 |
| 已审计常量或可信清洗器输出 | `原始` | 原样输出，调用方承担上下文责任。 |
| 已审计生成器 | `扩展` | 通过受预算约束的写入法原样输出。 |

“已经手工转义”不等于可信 HTML。编码依赖上下文，把属性编码后的文字放入标签、脚本、样式或 SVG 仍可能不安全。

## 标签策略

标签名必须匹配：

```text
^[A-Za-z][A-Za-z0-9:-]*$
```

以下标签按 ASCII 大小写不敏感拒绝：

```text
script style iframe object embed applet frame frameset base svg math
```

这阻止普通节点进入脚本、CSS、嵌入文档和未实现专用解析器的子语言上下文。自定义元素仍可使用；库不判断它是否已在浏览器注册。

## 属性策略

属性名必须匹配：

```text
^[A-Za-z_:][A-Za-z0-9_.:-]*$
```

以下属性按 ASCII 大小写不敏感拒绝：

- 所有以`on`开头的事件处理属性；
- `style`；
- `srcdoc`；
- `http-equiv`；
- `srcset`和`imagesrcset`；
- `ping`。

元素按大小写不敏感名称拒绝重复属性，避免解析器对`id`/`ID`或类似输入采用不一致值。

`data-*`、`aria-*`和自定义命名空间属性在满足名称语法时可使用。言页不会验证 ARIA 语义或自定义元素协议。

## URL 策略

以下属性进入 URL 校验：

```text
href src action formaction poster xlink:href cite background
longdesc usemap manifest profile
```

地址不得包含 UTF-8 字节值小于 33 的空白/控制字符或 DEL。若存在显式协议，只允许：

```text
http: https: mailto: tel:
```

没有显式协议的相对地址、根路径、查询和片段可使用。`javascript:`、`data:`和未知协议被拒绝。

此策略不验证：

- 主机、端口、DNS 或 IP 范围；
- 重定向后的最终地址；
- 同源、下载或导航授权；
- `mailto:`和`tel:`的业务允许范围；
- 服务端请求伪造边界。

应用应在生成节点前执行自己的地址与权限策略。

## 转义边界

`HTML文字节点`在渲染时使用标准模板库的 HTML 文字转义。文字属性也执行 HTML 转义并放在双引号内。调用方不得把未包装文字直接放入元素或片段。

言页不创建脚本、样式或 SVG 上下文，因此不声称提供这些上下文的编码器。需要这些能力时应使用专用库和独立安全审计。

## 原始与扩展

`HTML原始内容`/`原始`完全跳过转义。推荐范围：

- 源码内短小、固定、已审计的实体或片段；
- 另一个具有明确安全保证的结构化渲染器输出；
- 专用、上下文正确的 HTML 清洗器最终输出。

`HTML扩展节点`/`扩展`把受控写入法交给可信生成器。写出器不能取得内部状态或文本收集列，每个非空片段仍检查输出字符和片段预算。但它可以输出脚本、样式或任意标记，因此不能接收不可信值。

安全审查应搜索四个入口：

```text
HTML.原始（
HTML原始内容（
HTML.扩展（
HTML扩展节点（
```

资源预算不把不可信 HTML 变安全；它只限制工作量和输出量。

## 资源限制

默认预算：

| 维度 | 默认 | 硬上限 |
| --- | ---: | ---: |
| 深度 | 64 | 256 |
| 节点 | 10000 | 100000 |
| 每元素属性 | 256 | 1024 |
| 输出字符 | 4194304 | 16777216 |
| 输出片段 | 100000 | 1000000 |

预算覆盖普通节点、原始节点和扩展写出。深度限制同时约束递归遍历。直接子节点列和属性列在构造期也受硬上限保护。

字符按 Unicode 字符计数，不是字节。上层在 UTF-8 编码、压缩、排队或发送时应设置独立字节和时间限制。

## 流式失败

`流式渲染`在每个片段通过预算后立即调用外部写入法。回调错误或后续预算超限不会撤回已经写出的内容。

因此上层应：

- 在首个片段前完成认证、授权和状态码选择；
- 不在组件构建器中进行可能失败的远程读取；
- 对写入器设置超时、背压和取消；
- 为中途失败定义关闭连接或协议级错误尾部；
- 需要原子正文时改用`渲染受限`。

## 错误与信息泄露

稳定代码适合日志分类和程序分支。`错误详情`还包含源代码、类别、位置和踪迹，这些信息可能暴露内部文件结构或实现细节。

服务端不应把完整详情直接返回浏览器。对外响应使用应用级错误编号和通用消息；完整详情只进入受访问控制且已脱敏的诊断日志。

## 非目标与配套控制

言页 1.0 不提供：

- HTML 解析或富文本清洗；
- CSS URL 分析；
- JavaScript 安全执行；
- SVG/MathML 子语言策略；
- CSP、Trusted Types 或安全首部生成；
- Cookie、CSRF、会话或业务授权；
- DOM 或浏览器运行时隔离。

公开 Web 应用仍应使用 HTTPS、合适的 CSP、`X-Content-Type-Options`、安全 Cookie、CSRF 防护、输入大小限制和业务授权。

## 回归要求

安全变化至少覆盖：

- 文字与属性特殊字符；
- 标签和属性大小写变体；
- 控制字符与混淆协议；
- 所有 URL 属性；
- 重复属性；
- 空元素；
- 输入列和读取快照隔离；
- 原始与扩展预算；
- 深度、节点、属性、字符和片段边界；
- 结构化错误代码；
- 完整 HTML5 页面快照。

漏洞报告流程见仓库根目录 [SECURITY.md](../SECURITY.md)。
