# 言页使用指南

言页的默认路径只有两步：把数据放入有上下文的节点，再由统一渲染器输出。不要预先转义普通文字，也不要拼接标签字符串。

## 安装与导入

```sh
yanbao add html --package 言页 --version "^1.0"
```

```yanxu
引「包:言页」为 HTML；
```

言包会把稳定版本和内容校验写入`言序.lock`。CI 和生产恢复依赖时使用锁文件，不把分支头当成稳定依赖。

## 构造安全页面

```yanxu
定 标题：文 为 「<言序与 Web>」；

定 页面 为 HTML.文档（HTML.元素（「html」，【
    HTML.属性（「lang」，「zh-CN」）
】， 【
    HTML.元素（「head」，【】， 【
        HTML.元素（「meta」，【HTML.属性（「charset」，「utf-8」）】，【】），
        HTML.元素（「title」，【】，【HTML.文字（标题）】）
    】），
    HTML.元素（「body」，【】， 【
        HTML.元素（「h1」，【】，【HTML.文字（标题）】），
        HTML.元素（「a」，【HTML.属性（「href」，「/docs?q=1」）】，【
            HTML.文字（「阅读文档」）
        】）
    】）
】））；

言 页面.渲染（）；
```

`<`、`>`、`&`和引号在文字或属性上下文中自动转义。不要先手工转义，否则会产生重复编码。

属性值为`真`时输出布尔属性，为`假`或`空`时省略：

```yanxu
HTML.元素（「input」，【
    HTML.属性（「disabled」，真），
    HTML.属性（「hidden」，假）
】，【】）；
```

结果为`<input disabled>`。

## 片段和组件

片段组合多个同级节点，不增加包装标签：

```yanxu
定 导航 为 HTML.片段（【
    HTML.元素（「a」，【HTML.属性（「href」，「/」）】，【HTML.文字（「首页」）】），
    HTML.文字（「 · 」），
    HTML.元素（「a」，【HTML.属性（「href」，「/about」）】，【HTML.文字（「关于」）】）
】）；
```

组件延迟构建一个节点：

```yanxu
法 构建欢迎卡（） 则
    归 HTML.元素（「section」，【HTML.属性（「class」，「welcome」）】，【
        HTML.元素（「h2」，【】，【HTML.文字（「欢迎」）】），
        导航
    】）；
终

定 欢迎卡 为 HTML.组件（构建欢迎卡）；
```

构建器必须返回`HTML节点`，否则得到`HTML_COMPONENT`。需要参数的组件可以写成普通法，先根据参数返回节点，再把节点放入页面。

## 快照隔离

元素和片段复制构造时传入的列：

```yanxu
令 子项：列 为 【HTML.文字（「甲」）】；
定 容器 为 HTML.元素（「div」，【】，子项）；

追加（子项，HTML.文字（「不会进入容器」））；
```

`各属性（）`和`各子节点（）`也返回新列。修改读取结果不会改变节点。需要变更节点时使用`添属性`或`添子`，并把构建期可变性限制在单一所有者中。

## 受限完整渲染

默认限制适合一般页面。处理多租户模板或外部数据时应按业务负载收紧：

```yanxu
定 正文：文 为 HTML.渲染受限（页面，{
    「最大深度」：32，
    「最大节点」：5000，
    「最大属性」：64，
    「最大输出字符」：1048576，
    「最大片段」：50000
}）；
```

字典可以只给出需要覆盖的项。`规范限制`会补齐默认值，并拒绝未知键、小数或超出硬上限的配置。

## 增量渲染

`流式渲染`直接把每个非空片段交给同步回调，不先联结完整正文：

```yanxu
令 各片：列<文> 为 【】；

法 收片（片：文）：空 则
    追加（各片，片）；
    归 空；
终

定 统计：典 为 HTML.流式渲染（页面，收片，{
    「最大输出字符」：1048576
}）；
```

联结`各片`得到的正文与相同限制下的`渲染受限`一致。统计按实际写入返回`节点数`、`字符数`和`片段数`。

流式路径不是事务：若回调抛错，或后续节点超过预算，先前片段已经离开渲染器。上层 HTTP 实现应在写首部前完成认证、授权和数据读取，并为中途失败定义关闭或尾部错误策略。

## 错误处理

抛出式路径：

```yanxu
试 则
    言 HTML.安全渲染（页面）；
救 所误 则
    定 详情：典 为 HTML.错误详情（所误）；
    若 （详情【「代码」】 等于 「HTML_LIMIT_OUTPUT」） 则
        言 「页面超过输出预算」；
    终
终
```

结果式路径：

```yanxu
定 结果：典 为 HTML.尝试渲染（页面）；

若 结果【「成功」】 则
    言 结果【「值」】；
不然
    言 结果【「错误」】【「代码」】；
终
```

只按`代码`分支；`消息`可能在补丁版本中改进措辞。

## 可信输出

`原始`适合短的已审计常量：

```yanxu
定 图标实体 为 HTML.原始（「&#9733;」）；
```

`扩展`适合结构化生成器，它只能通过渲染器提供的写入法输出：

```yanxu
法 写徽标（写入法：法）：空 则
    写入法（「<span class=\"badge\">」）；
    写入法（「已审计常量」）；
    写入法（「</span>」）；
    归 空；
终

定 徽标 为 HTML.扩展（写徽标）；
```

两者均不清洗输入。不要把用户、数据库、文件或网络内容交给它们；需要接收富文本时，先使用独立且适合目标上下文的 HTML 清洗器，再把其输出视为可信数据。

## 与 Web 层集成

言页不申请监听或网络权限，也不决定 HTTP 首部。上层服务应：

- 以 UTF-8 发送正文；
- 使用`text/html; charset=utf-8`；
- 在应用层设置 CSP、`X-Content-Type-Options`和安全 Cookie；
- 在流式发送前确定状态码和不可恢复错误策略；
- 对导航目标、重定向、下载和服务端请求另做主机与业务授权。

安全策略详见 [SECURITY_MODEL.md](SECURITY_MODEL.md)，所有签名详见 [API.md](API.md)。
