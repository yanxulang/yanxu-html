# 迁移到言页 1.0

本指南面向从 `yanxu-html` 0.1.x 升级到言页 1.0.0 的应用。1.0 收紧默认安全策略并采用稳定中文包名，因此升级前应运行页面快照和拒绝路径测试。

## 1. 更新工具链

使用言序 1.1.12 或更高的兼容 1.x 版本，并让清单保持格式 2。

```toml
[包]
格式 = 2
言序 = ">=1.1.12"
```

## 2. 更新依赖与导入

0.1.x 常见清单：

```toml
[依赖]
yanxu-html = { git = "https://github.com/yanxulang/yanxu-html.git", 修订 = "main", 版 = "^0.1" }
```

1.0 稳定清单：

```toml
[依赖]
言页 = { git = "https://github.com/yanxulang/yanxu-html.git", 修订 = "v1.0.0", 版 = "^1.0" }
```

把导入从：

```yanxu
引「包:yanxu-html」为 HTML；
```

改为：

```yanxu
引「包:言页」为 HTML；
```

然后串行更新并离线验证锁文件：

```sh
yanxu 包 更新 .
yanxu 包 锁 --离线 .
```

不要把最终稳定依赖固定到`main`或发布分支。

## 3. 处理更严格的安全策略

1.0 的普通`元素`拒绝：

```text
script style iframe object embed applet frame frameset base svg math
```

普通`属性`拒绝所有`on*`事件属性，以及：

```text
style srcdoc http-equiv srcset imagesrcset ping
```

同一元素的属性名按 ASCII 大小写不敏感比较，`id`与`ID`视为重复。以前依赖这些上下文的代码不能通过字符串拼接恢复；应把脚本、样式、SVG 或复杂响应图交给专用、可审计的生成与清洗边界。

## 4. 复查 URL 属性

1.0 对以下属性执行相同 URL 策略：

```text
href src action formaction poster xlink:href cite background
longdesc usemap manifest profile
```

显式协议只允许`http:`、`https:`、`mailto:`和`tel:`。相对地址、根路径、查询和片段仍可使用。含空白、控制字符、DEL、`data:`、`javascript:`或未知协议的值会返回`HTML_URL`。

主机白名单、重定向、下载目标和服务端请求仍由应用验证。

## 5. 采用结构化错误

不要继续按错误消息文字判断：

```yanxu
试 则
    HTML.安全渲染（页面）；
救 所误 则
    定 详情：典 为 HTML.错误详情（所误）；
    若 （详情【「代码」】 等于 「HTML_TAG_UNSAFE」） 则
        # 记录或返回应用级错误。
    终
终
```

需要结果式接口时使用`尝试渲染`。完整代码表见 [API.md](API.md)。

## 6. 设置资源预算

0.1.x 没有统一可配置预算。1.0 默认限制为 64 层、10000 节点、每元素 256 属性、4194304 输出字符和 100000 输出片段。

若页面由租户配置、递归数据或大型集合生成，应显式收紧：

```yanxu
定 正文 为 HTML.渲染受限（页面，{
    「最大深度」：32，
    「最大节点」：5000，
    「最大输出字符」：1048576
}）；
```

组件和片段本身也计入节点与深度。迁移时用真实最大页面验证，不要只按元素标签数估算。

## 7. 选择完整或流式输出

现有`渲染`和`安全渲染`保持完整正文语义。新`流式渲染`可避免在库内收集完整正文，但已写出的片段不会因后续错误回滚。

从完整响应迁到流式 HTTP 前，应明确：

- 认证、授权和状态码何时确定；
- 写入回调失败如何关闭响应；
- 字符预算外是否还需要编码后的字节预算；
- 上层队列如何实现背压与取消。

## 8. 复查可信接口

`原始`继续存在以保持显式可信内容兼容；它没有变成清洗器。新`扩展`适合多片可信生成器。两者都会进入字符与片段预算，但都不会自动转义内容。

升级时全局搜索：

```text
HTML.原始（
HTML原始内容（
HTML.扩展（
HTML扩展节点（
```

确认输入只来自已审计常量、结构化可信生成器或专用清洗器的输出。

## 9. 回归清单

- 更新包名、版本范围、稳定标签和锁文件；
- 静态检查所有消费者；
- 运行完整页面快照，特别是 doctype、空元素、属性顺序和转义；
- 覆盖被拒绝标签、属性、URL 和大小写重复项；
- 覆盖组件错误与资源超限代码；
- 验证最长正常页面在新预算内；
- 若使用流式输出，模拟中途写入失败；
- 审计全部`原始`与`扩展`调用；
- 在实际 HTTP 层验证 UTF-8、内容类型、CSP 和安全首部。

完成后把应用依赖固定在 1.0 标签和锁文件，不保留临时路径或分支依赖。
