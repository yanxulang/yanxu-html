# 兼容性

## 工具链

| 项目 | 1.0 边界 |
| --- | --- |
| 言序 | `>=1.1.12` |
| 言包清单 | 格式 2 |
| 包名 | `言页` |
| 版本范围 | `^1.0` |
| 第三方依赖 | 无 |

仓库根包、示例和基准锁由言序 1.1.12 生成并支持离线校验。后续言序 1.x 是兼容目标；未进入 CI 或消费者验证的版本不列为已验证。

## 平台

言页核心为纯言序实现，不加载原生制品，也不依赖操作系统命令、文件布局或网络服务。

| 环境 | 1.0 支持边界 | 发布验证 |
| --- | --- | --- |
| macOS ARM64 | 言序 1.1.12 字节码 VM | 源码、规格、示例、基准、Release YXB |
| Linux x86-64 | 言序 1.1.12 字节码 VM | GitHub Actions 全门禁 |
| Windows x86-64/ARM64 | 言序 1.1.12 字节码 VM | 纯语言兼容目标；本发布不声称独立运行验证 |

应用仍应在自己的 HTTP 写入器、字符编码、代理和浏览器安全策略上运行集成测试。

## HTML 输出契约

- 输出文字为言序`文`，HTTP 层应按 UTF-8 发送；
- `HTML文档`固定以`<!doctype html>\n`开头；
- 元素与属性保持调用方给定的名称大小写和顺序；
- 普通文字和文字属性按言序标准模板库的 HTML 规则转义；
- 布尔属性为`真`时只输出名称，为`假`或`空`时省略；
- 空元素不输出闭合标签，并拒绝子节点；
- 片段本身不增加包装标记；
- `流式渲染`各片按最终正文顺序同步写出，联结后与`渲染受限`结果一致。

以上语义在 1.x 内保持兼容。输出片段的具体分片边界不是跨主版本的序列化格式；消费者应依赖联结后的内容和统计，而不要把某个标签固定为一次回调。

## 权限

稳定清单固定为无宿主权限：

```text
文件 = []
网络 = []
TCP监听 = []
UDP绑定 = []
环境 = []
进程 = false
原生扩展 = false
```

言页不会替 HTTP 服务申请监听或网络权限。应用使用`yanxu-web`或其他服务器时，由顶层应用声明实际权限。

## 1.x 兼容承诺

1.x 保持公开类、法名、参数含义、错误代码、默认预算键、默认安全拒绝项和可信接口语义。兼容版本可以新增工厂、可选限制或更严格的输入诊断，但不会静默接受以前拒绝的危险上下文。

以下变化需要新的主版本：

- 删除或重命名公开入口；
- 改变普通文字/属性的转义语义；
- 放宽脚本、样式、嵌入、事件或 URL 协议策略；
- 让`原始`或`扩展`绕过资源预算；
- 改变结构化错误代码或结果字段；
- 降低默认资源限制并破坏既有正常负载。

0.1.x 到 1.0.0 的迁移见 [docs/MIGRATION.md](docs/MIGRATION.md)。
